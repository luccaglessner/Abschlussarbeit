Ausführungsreihenfolge der Jupyter Notebooks
=============================================

Dieses Dokument beschreibt die empfohlene Reihenfolge der Ausführung aller Jupyter Notebooks in diesem Projekt. Diese Reihenfolge stellt sicher, dass alle benötigten Input-Dateien für nachfolgende Notebooks korrekt generiert wurden.

Automatisierte Pipelines (in `Jupyter Notebooks/Pipeline_Scripts/`):
-> run_pipeline_1.py   [Führt Schritt 1: Akquise aus]
-> run_pipeline_2_3.py [Führt Schritt 2 & 3: Analyse & Machine Learning aus]
-> run_pipeline_4.py   [Führt Schritt 4: Imputation (Training -> Inferenz -> Evaluation) aus]
-> run_pipeline_All.py [Führt ALLE Pipelines sequenziell aus]

--------------------------------------------------------------------------------------------------
0. 0_Überprüfung_geeigneter_Datenbanken
   - Pfad: `0_Überprüfung_geeigneter_Datenbanken/Überprüfung_geeigneter_Datenbanken.ipynb`
   - Zweck: Überprüft die Verfügbarkeit und Struktur der ursprünglichen Rohdaten.
   - Abhängigkeit: Keine (Liest Rohdaten).
   - Ausführung: Optional / Als erster Schritt zur Überprüfung.

1. Akquise
--------------------------------------------------------------------------------------------------

1. 1_Acquisition/1.1_Data-Acquisition-Wrapper (KRITISCH)
   - Pfad: `1_Acquisition/1.1_Data-Acquisition-Wrapper/Data-Acquisition-Wrapper.ipynb`
   - Zweck: Liest die Rohdaten aus `Gesammelte_Datenbanken` ein, homogenisiert sie und erstellt die "Komplette Datenbank".
   - Output: Erzeugt `Angepasste_Datenbanken/Komplette_Datenbank/.../Komplette_Datenbank.csv`.
   - Wichtig: Dieses Notebook muss zwingend ausgeführt werden, bevor andere Schritte (außer 0) durchgeführt werden können.

2. Analytik
--------------------------------------------------------------------------------------------------

2. 2_Analysis/2.1_Explorative-Datenanalyse
   - Pfad: `2_Analysis/2.1_Explorative-Datenanalyse/Data_Exploration.ipynb`
   - Zweck: Allgemeine explorative Analyse der Daten.

3. 2_Analysis/2.2_Rock-Type_Analysis [Verwendung nur für "without_added_rock-types"]
   - Pfad: `2_Analysis/2.2_Rock-Type_Analysis/Rock-Type_Analyzer.ipynb`
   - Zweck: Vergleicht die Datenqualität und -verteilung vor und nach dem Rock-Type Mapping.
   - Abhängigkeit: Benötigt Outputs von 1.1 UND 1.2.

4. 2_Analysis/2.3_Temperature_Analysis
   - Pfad: `2_Analysis/2.3_Temperature_Analysis/Temperature_Analysis.ipynb`
   - Zweck: Analyse der Temperaturverteilungen.

5. 2_Analysis/2.4_Data-Quality_Ionic-Balance-Error
   - Pfad: `2_Analysis/2.4_Data-Quality_Ionic-Balance-Error/...`
   - Zweck: Analyse des Ionenbilanzfehlers.
   - Ablauf innerhalb der Ordner:
     a. `1.1_Data-Quality_Raw_Data.ipynb`
     b. `1.2_Data-Quality_Adjusted_Data.ipynb`

3. Machine Learning Pipeline (Cluster-Analysen)
--------------------------------------------------------------------------------------------------

6. 3_Machine-Learning/3.1_Preprocessing (HAUPTVORAUSSETZUNG FÜR ML)
   - Pfad: `3_Machine-Learning/3.1_Preprocessing/Preprocessing.ipynb`
   - Zweck: Bereitet die Daten für das Machine Learning vor (Log-Transformation, Scaling).
   - Input: Liest `Komplette_Datenbank.csv`.
   - Output: Erzeugt `Preprocessing/.../Preprocessed_SOM_Ready.csv`.

7. 3_Machine-Learning/3.2_Machine-Learning
   - Pfad: `3_Machine-Learning/3.2_Machine-Learning/MiniSom/MiniSom_Machine-Learning.ipynb`
   - Pfad: `3_Machine-Learning/3.2_Machine-Learning/IntraSom/IntraSom_Machine-Learning.ipynb`
   - Zweck: Trainiert Self-Organizing Maps (SOM) zur Cluster-Analyse.
   - Input: Liest `Preprocessed_SOM_Ready.csv`.

4. Imputation (VAE)
--------------------------------------------------------------------------------------------------

8. 4_Imputation/4.1_VAE_Imputation
   - Pfad: `4_Imputation/4.1_VAE_Imputation/VAE_Imputation.ipynb`
   - Zweck: Trainiert das Variational Autoencoder (VAE) Modell.
   - Output: Speichert trainierte Modelle.

9. 4_Imputation/4.2_Inference
   - Pfad: `4_Imputation/4.2_Inference/Inference.ipynb`
   - Zweck: Nutzt das trainierte VAE-Modell zur Imputation fehlender Werte in den Daten.
   - Output: Generiert Imputations-Ergebnisse.

10. 4_Imputation/4.3_Evaluation
    - Pfad: `4_Imputation/4.3_Evaluation/Evaluation.ipynb`
    - Zweck: Erstellt Berichte und Diagramme zur Qualität der Imputation.
    - Output: `Evaluation_Report.pdf`.
